# Retrospective: Epic 1 - Capture

**Date:** 2026-04-22

## Epic Summary

Epic 1 successfully delivered the core "Capture" experience. Users can now add todos via a persistent input, view them in a reactive list, and benefit from global error handling and local persistence.

### Delivery Metrics
- **Stories Completed:** 6/6 (100%)
- **Test Coverage:** 92.53% (src/lib/), 79 Unit Tests, 20 E2E Tests
- **Velocity:** High (All work delivered in the initial sprint)

## Successes & Strengths
- **Minimalist UX:** The 'Silence is a Feature' philosophy worked perfectly for onboarding.
- **Data Robustness:** Dexie + useLiveQuery provides seamless, reactive local storage.
- **Architectural Discipline:** Strict adherence to SSR guards and schema invariants.
- **Testing Standard:** Establishing E2E coverage from the first story provided a massive safety net.

## Challenges & Growth Areas
- **Tailwind v4 Migration:** Initial friction adapting to the CSS-first token system.
- **Test Environment:** Managing jsdom limitations for IndexedDB and matchMedia.
- **SSR Boundaries:** Constant vigilance required to keep client-only logic out of server components.

## Key Insights
1. **Upfront hardening works:** Spending time on Zod and Dexie early prevented data-corruption bugs later.
2. **CSS-First is better:** Once adapted, Tailwind v4's CSS tokens are more readable than JSON-style configs.
3. **E2E is Mandatory:** The capture-flow logic was refined through Playwright assertions.
